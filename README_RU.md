# MP3 Library to Zvuk.com Comparator

Сравнение вашей локальной MP3-библиотеки с коллекцией на музыкальном сервисе zvuk.com.

## Возможности

- 📁 Сканирование локальной MP3-библиотеки с извлечением метаданных (ID3-теги)
- 🎵 Получение вашей коллекции из zvuk.com через GraphQL API
- 🔍 Сравнение библиотек и поиск различий
- 🧠 Умное сопоставление треков с нормализацией:
  - Алиасы исполнителей (t.A.T.u. = Тату = tatu)
  - Многоавторские треки (Placebo feat. Alison Mosshart = Placebo)
  - Конвертация чисел (30 Seconds to Mars = THIRTY SECONDS TO MARS)
  - Удаление приставки "The" (The Beatles = Beatles)
  - Варианты имён (Цой = Виктор Цой)
  - Очистка названий (номера треков, OST, ремастеры)
- 📊 Экспорт результатов в CSV-файлы
- 💻 Вывод в консоль с краткой и подробной информацией

## Установка

### Требования

- Python 3.10 или выше

### Установка зависимостей

```bash
pip install -r requirements.txt
```

## Использование

### Базовое сравнение библиотек

```bash
python main.py compare --token ВАШ_ТОКЕН --library-path /путь/к/вашей/мп3/библиотеке
```

### Получение API токена

**Анонимный токен (ограниченный):**
```bash
python -c "from src.zvuk_api import ZvukAPIClient; print(ZvukAPIClient.get_anonymous_token())"
```

**Персональный токен (рекомендуется):**
1. Войдите на https://zvuk.com
2. Откройте DevTools браузера (F12)
3. Перейдите на вкладку Network
4. Посетите https://zvuk.com/api/tiny/profile
5. Скопируйте значение поля `token` из ответа

## Команды

### `compare` — Сравнение библиотек

```bash
python main.py compare [ОПЦИИ]

Опции:
  -t, --token TEXT        API токен Zvuk.com (обязательно)
  -l, --library-path PATH Путь к локальной MP3-библиотеке (обязательно)
  -o, --output PATH       Каталог для CSV-файлов
  --test-connection       Проверить подключение к API и выйти
  -v, --verbose           Подробный вывод
  -q, --quiet             Тихий режим (только ошибки)
```

**Примеры:**

```bash
# Сравнить библиотеки
python main.py compare -t ТОКЕН -l "D:\Music" -o output

# Сначала проверить подключение
python main.py compare -t ТОКЕН -l "D:\Music" --test-connection

# Подробный вывод
python main.py compare -t ТОКЕН -l "D:\Music" -v
```

## Вывод

### Результаты сравнения

В выходном каталоге создаются три CSV-файла:

| Файл | Описание |
|------|----------|
| `comparison_result_only_local.csv` | Треки только в вашей локальной библиотеке |
| `comparison_result_only_zvuk.csv` | Треки только в коллекции zvuk.com |
| `comparison_result_match.csv` | Треки, присутствующие в обеих библиотеках |

Каждый CSV содержит следующие колонки:
- `title` — Название трека
- `artist` — Исполнитель
- `album` — Альбом
- `year` — Год выпуска
- `genre` — Жанр
- `duration` — Длительность в секундах
- `file_path` — Путь к файлу (для локальных треков)
- `zvuk_id` — ID трека в zvuk.com (для треков из zvuk)

Приложение использует GraphQL API zvuk.com по адресу `https://zvuk.com/api/v1/graphql`

## Решение проблем

### "Configuration error: Path does not exist"

Проверьте, что `--library-path` указывает на существующий каталог.

### Ошибка подключения к API

- Проверьте, что токен действителен
- Проверьте подключение к интернету
- Попробуйте получить новый токен

### CSV-файлы пустые

- Проверьте, что в локальной библиотеке есть MP3-файлы
- Убедитесь, что путь к библиотеке указан верно
- Запустите с флагом `-v` для подробного логирования

### Ошибки кодировки в CSV

Приложение использует UTF-8 с BOM для корректного отображения кириллицы в Excel. Если возникают проблемы:
- Откройте CSV в Excel через «Данные» → «Из текста/CSV»
- Выберите кодировку UTF-8
- Разделитель: точка с запятой (;)
